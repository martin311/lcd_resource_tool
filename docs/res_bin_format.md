# LRES res.bin MVP Format

`res.bin` is a little-endian resource package generated by:

```bash
python tools/main.py pack --target esp32s3
```

The current MVP only packs image resources converted from
`database/resource_manifest.json`.

## Header

Packed with Python `struct` format `<4sIIIII`.

| Field | Type | Description |
| --- | --- | --- |
| `magic` | `char[4]` | Always `LRES` |
| `version` | `uint32` | Format version, currently `1` |
| `entry_count` | `uint32` | Number of resource entries |
| `index_offset` | `uint32` | Byte offset of the entry table |
| `data_offset` | `uint32` | Byte offset of the first resource payload |
| `file_size` | `uint32` | Total package size in bytes |

Header size is 24 bytes.

## Entry

Packed with Python `struct` format `<IHHHHIIII`.

| Field | Type | Description |
| --- | --- | --- |
| `resource_id_hash` | `uint32` | `zlib.crc32(resource_id.encode("utf-8"))` |
| `type` | `uint16` | Resource type code |
| `format` | `uint16` | Resource format code |
| `width` | `uint16` | Image width |
| `height` | `uint16` | Image height |
| `offset` | `uint32` | Absolute byte offset of resource payload in `res.bin` |
| `compressed_size` | `uint32` | Stored payload size in bytes |
| `raw_size` | `uint32` | Raw RGB565 size before compression |
| `crc32` | `uint32` | `zlib.crc32()` of the stored payload |

Entry size is 28 bytes.

## Codes

Resource type codes:

| Code | Name |
| --- | --- |
| `1` | image |

Image format codes:

| Code | Name |
| --- | --- |
| `1` | raw_rgb565 |
| `2` | rgb565_rle |

## Layout

```text
0x0000  header
        entry[0]
        entry[1]
        ...
        resource payload bytes
```

The generated `output/<target>/resource_table.json` mirrors the binary index in
a human-readable form for debugging and later firmware integration.
